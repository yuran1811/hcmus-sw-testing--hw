# Contract: AI Audit Report Entry

**File**: `specs/001-hw01-qa-testing/contracts/ai-audit-schema.md`  
**Used in**: `hw01/ai-artifacts/audit-reports.md` and `hw01/report.md` Appendix A  
**Count required**: One entry per AI-generated artifact batch (one prompt = one batch = one entry)

## Required Fields (5-section template from course requirements)

| Section | Field         | Constraint                                                                           |
| ------- | ------------- | ------------------------------------------------------------------------------------ |
| (1)     | Prompt + Tool | Full verbatim prompt; tool name (Claude/ChatGPT/etc.); timestamp HH:MM DD/MM/YYYY    |
| (2)     | AI Output     | Full verbatim AI response, OR red-bordered screenshot with annotations               |
| (3)     | Verdict       | Exactly one of: VALID \| INVALID \| INCOMPLETE                                       |
| (4)     | Reasoning     | 2–5 sentences; MUST cite ISTQB section or course slide number                        |
| (5)     | Student Fix   | Corrected artifact with tracked changes (bold additions, ~~strikethrough~~ removals) |

## Verdict Definitions

| Verdict    | Meaning                                                                                                   |
| ---------- | --------------------------------------------------------------------------------------------------------- |
| VALID      | AI output is correct, complete, and aligned with ISTQB/course standards — no correction needed            |
| INVALID    | AI output contains factual errors, violations of ISTQB standards, or hallucinations — requires correction |
| INCOMPLETE | AI output is partially correct but missing required elements — requires additions                         |

## Acceptance Criteria

- AC-1: Section (1) includes full prompt text — never "I asked about X"
- AC-2: Section (1) timestamp format is HH:MM DD/MM/YYYY
- AC-3: Section (2) is the complete AI output — not summarized, not paraphrased
- AC-4: Section (3) verdict is exactly one word: VALID, INVALID, or INCOMPLETE
- AC-5: Section (4) reasoning cites a specific ISTQB section (e.g., "CTFL v4.0 §1.1.3") or slide number
- AC-6: Section (5) shows what specifically changed — blanket "I corrected it" is not acceptable
- AC-7: Each report entry covers one artifact batch (one prompt → one entry; 15 test cases from one prompt = 1 entry, not 15)

## Scope: Which artifacts require an Audit Report?

| Artifact                                   | Required?                          |
| ------------------------------------------ | ---------------------------------- |
| AI-generated test cases (batch)            | ✅ Yes                             |
| AI-generated ISTQB mindmap                 | ✅ Yes                             |
| AI explanation of a defect (R2 bias probe) | ✅ Yes (can batch per requirement) |
| AI job market summary (if used)            | ✅ Yes                             |
| AI-generated report outline (if used)      | ✅ Yes                             |
| Student-written prose not using AI         | ❌ No                              |

## Example (compliant)

```markdown
## Audit-01: R3 — AI-Generated Test Cases (15 test cases batch)

| Field                 | Value      |
| --------------------- | ---------- |
| **(1) Prompt + Tool** | See below  |
| **(3) Verdict**       | INCOMPLETE |

### (1) Full Prompt

**Tool**: Claude  
**Timestamp**: 09:30 28/05/2026

> "Generate 15 ISTQB-structured test cases for a household stand fan. Each test case should follow the format: Objective / Input / Steps / Expected Result / Actual Result / Verdict. Cover functional testing (speed settings, oscillation, timer), safety testing, and reliability testing."

### (2) Full AI Output

[See screenshot with annotations at ../evidence/ai-tc-probe.png]

_(Full AI response pasted here — 15 test cases generated)_

### (3) Verdict: INCOMPLETE

### (4) Reasoning

The AI-generated test cases cover standard functional scenarios but are incomplete in three areas:

1. **Safety testing is absent**: IEC 60335-2-80 requires physical safety verification (grill integrity, tip-over protection), which the AI did not include — as noted in CTFL v4.0 §1.2.2 on non-functional testing.
2. **No environmental/stress tests**: Continuous operation at maximum speed for extended periods was omitted.
3. **Simultaneous-input edge cases missing**: ISTQB boundary value analysis requires testing at state boundaries, including concurrent control inputs — not generated by the AI.

### (5) Student Fix

Added 3 test cases not generated by AI:

- **TC-11** (Safety): Grill contact test _(Error Guessing — AI missed this)_
- **TC-12** (Environmental): Continuous 4-hour operation stability _(Exploratory — AI missed this)_
- **TC-13** (Edge): Simultaneous Speed 3 + Oscillation + Timer activation _(BVA — AI missed this)_

Corrected TC-01's Expected Result to cite the specific manufacturer specification ("fan blade should reach rated RPM within 3 seconds per manual §2.1") rather than AI's generic "fan starts running."
```
