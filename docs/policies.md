# Assignment Policies

## General Principles & Core Criteria

- **Individual Work:** All assignments in this course are individual assignments. Students are encouraged to discuss with lecturers, teaching assistants, classmates, and utilize AI tools effectively and responsibly during the assignment process.
- **Course Criteria:** The core criteria of this Software Testing course are the smart and effective utilization of AI, and the methodical execution of testing processes following correct techniques. The number of test cases and bugs discovered will serve as a basis for grade differentiation among students.
- **AI Disclosure:** Students must declare their AI usage in accordance with the "AI Usage Guideline" document as well as specific instructions from the lecturers.

## File Format & Presentation Requirements

- **Text-Based Formats:** All assignment submissions must be written in text-based files. The use of Markdown is highly encouraged (LaTeX is also acceptable).
- **Format Transitions:**
  - If students previously used **Word**, they must switch to submitting in Markdown format.
  - If students use **Excel**, they may continue working on Excel, but the resulting summary tables must be copied into the Markdown file.
  - If students present via **PowerPoint**, they are encouraged to write in Markdown using presentation mode.
- **PDF Copies:** In addition to submitting the Markdown/LaTeX files, students must also submit a "Save-As-PDF" version of those files.
- Submission file name: `StudentID_ExerciseID_SelfAssessedGrade.zip`

## Version Control Requirements (Git)

- **Git Usage:** Students must use Git (GitHub / Bitbucket / GitLab) to manage the version history of their assignment files.
- **Commit Messages:** Each assignment consists of several requirements, and each requirement involves multiple steps. For every step within a requirement, students must create a clear and explicit Git commit message.
- Students may work on their personal / group GitHub repositories and **submit the git commit logs**.

## Project Collaboration & Peer Review Guidelines

- **Work Allocation:** For project-related assignments, although each student will execute their own specific requirements, they must ensure their work does not overlap with other team members.
- **Task Complexity:** Students must balance the significance and difficulty of tasks within the group. Choosing tasks that are too simple will limit the maximum achievable grade, even if the work is fully completed.
- **Peer Reviews:** Students are required to cross-review the work of their team members. The goal of the review, similar to software testing, is to find constructive feedback for teammates to refine their work. Reviews that only offer praise without constructive critique will not be highly valued.

## Self-Assessment & Grading Policy

- **Self-Assessment:** Each assignment includes a self-assessment rubric. Students are required to perform this self-assessment and grade their own work.
- **File Naming Format:** Submissions must be zipped and named strictly according to the format: `StudentID_ExerciseID_SelfAssessedGrade.zip`. The `SelfAssessedGrade` must be a 3-digit number ranging from `000` to `100`.
- **Grading Weights:** When the instructors grades the assignments, certain exercises/tasks will carry a higher weight than others.

## Submission Guidelines & Deadlines

- **Submission Platform:** All assignments must be submitted via Moodle.
- **Late Submissions:** Late submissions are strictly not accepted. Exceptions are only granted for important reasons (e.g., critical health or family emergencies), which must be notified to the instructors via email with full supporting evidence.
- **File Restrictions:** The submission link accepts a maximum of 20 files, with each file limited to 20 MB. Students should proactively use the "split-and-zip" feature on compression tools.
- **Online Links Policy:** Misuse or over-reliance on online links in the submission will result in a score of zero (0) for the assignment.

## Compliance & Academic Integrity

- **Strict Compliance:** All regulations must be strictly followed. Any violation of these submission guidelines will result in a score of 0 for the assignment.
- **Anti-Cheat Policy:** Any violation of the Anti-Cheat regulations will be treated as academic dishonesty/cheating in the course.
